背景

为什么需要可学习的物理模拟器？

🌊 传统方法的局限

  • 基于偏微分方程（PDE），计算代价昂贵
  • 需要精细调节物理参数，泛化性有限
  • 难以处理大规模粒子系统
  • 不支持端到端可微分学习

🤖 GNS 的优势

  • 从数据中学习物理规律，无需手工建模
  • 图神经网络表达粒子交互
  • 推理速度比传统方法快数十倍
  • 一个框架统一水、沙、胶等材料

🎯 论文目标

  • 提出 通用粒子模拟学习框架
  • 支持跨材料（水 / 沙 / 胶 / 布料）泛化
  • 在 6 种物理场景中 SOTA 性能

论文概览

Learning to Simulate Complex Physics with Graph Networks

📌 核心思想

将物理系统抽象为粒子图（Particle Graph）， 每个粒子为节点，邻近粒子间的交互为边。 通过图神经网络进行消息传递（Message Passing）， 预测每个粒子下一步的加速度，再积分得到位置。

🔬 输入特征

  • 节点：过去 N 帧速度 + 到边界距离 + 粒子类型嵌入
  • 边：相对位移 + 粒子间距离
  • 本复现：历史帧数 N_HIS = 6，边维度 3

🔁 训练目标

预测归一化加速度（对速度噪声鲁棒）， 均方误差损失（MSE），仅对动态粒子计算损失（排除静态边界粒子）。

📊 评估指标

滚动位置 MSE（Rollout Position MSE）： 从初始帧出发自回归预测完整轨迹， 与真实轨迹逐帧比较位置误差。

图构建

KDTree 半径搜索（r = 0.015），每粒子最多 150 条边，包含自环

网络规模

隐藏层维度 128，GNN 层数 10，粒子类型嵌入 16

训练设置

学习率 1e-4，指数衰减，添加噪声 σ=6.7e-4 提升鲁棒性

复现实现

GNS-PyTorch：纯 PyTorch 原生实现

🐍 技术栈

  • Python 3.10 + PyTorch 2.5
  • CUDA 12.1，支持 CPU fallback
  • 纯 PyTorch 原生 API，无 TF 依赖
  • TensorBoard 训练日志

📁 项目结构

  • models/ — 图网络层 + 动力学模型
  • datasets/ — TFRecord 转换 & 数据加载
  • trainer.py — 训练循环 + LR 调度
  • evaluator.py — 滚动推理 + GIF 生成

⚙️ 数据流程

  • 下载 WaterRamps TFRecord 数据集
  • extract_tfrs.py 转为 PyTorch 格式
  • 在线构建图（每步 KDTree 搜索）
  • 动态粒子 mask 过滤边界粒子

🎬 评估输出

  • 自回归滚动 601 步 预测完整轨迹
  • 生成预测 vs 真实对比动图 (GIF)
  • 每 25,000 步保存检查点并验证
